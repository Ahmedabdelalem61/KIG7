Small glossary

These words appear often in Odoo HR and in this project.

WordSimple meaning
EmployeeA person who works for the company. They have a file in the system.
ContractThe legal agreement for work: dates, wage, job.
Master dataImportant fixed lists and fields you use everywhere (for example: rank, visa type, project).
PayslipA paper or PDF that shows pay for one period.
Salary structureThe rules that build the payslip (base pay, allowances, deductions).
ExpenseMoney the employee or company spent (here used for flight tickets).
ModuleA package of features you install in Odoo.

Start: install the application

Odoo module: hr_uae_app — “HR UAE Admin”

Purpose

You install one application. It pulls in the full UAE HR platform. You do not need to install each small module by hand.

What you do

  • Open Odoo Apps.
  • Search for HR UAE Admin.
  • Click Install.

What you get

After install, the menu tree includes: Master Data, Documents, Audit, Leaves, Flights, Payroll, Adjustments, Termination, Reports, and the backend theme stack (see Phase 11).

Tip. Use a test database first. Train HR users there before you use production data.

Phase 1 — UAE base (foundation)

hr_uae_base — “HR UAE - Base”

Purpose

This layer sets the country context for UAE HR: money type (currency), working time rules, public holidays, and who can see which menus.

What you set up

  • Check that currency and working calendar match your company policy.
  • Review public holidays data for the year you operate.
  • Assign security groups to users (who is HR officer, who is manager, etc.).

What HR does later

HR uses the root menu to open other phases. Users only see what their group allows.

Link to next phase

When base data is correct, you continue to employee master data. That is where each person gets UAE-specific fields.

Phase 2 — Employee master data

hr_uae_master_data

Purpose

This is the single place for important employee facts for UAE operations: rank, passport, roster, position, project, cost center, employment status, visa.

What you set up

  • Define ranks and positions your company uses.
  • Set up analytic accounts / plans if you report cost by project or cost center.
  • Align fields with your contract process so wage and job data stay consistent.

What HR does every day

  • Create or open an employee.
  • Fill UAE fields (visa, passport, project, roster, and so on).
  • Keep status up to date (for example: active, on leave).

Link to next phase

When employee facts exist, you attach documents (passport scan, visa, medical). You also use this data later in payroll and reports.

Phase 3 — Employee documents

hr_uae_documents

Purpose

You store typed files per employee: passport, visa, medical, photo, contract. The system can warn you before a document expires.

What you set up

  • Decide who may see which documents (record rules protect owners).
  • Turn on mail templates if you want email alerts for expiry.

What HR does

  • Upload or renew documents on the employee form.
  • React to expiry alerts so visas and medical cards stay valid.

Link to next phase

Documents support compliance. Audit trail (next) shows who changed important HR values over time.

Phase 4 — Audit trail

hr_uae_audit_trail

Purpose

You can prove what changed, old value, new value, who changed it, and when. This helps internal control and disputes.

What you set up

  • Give access only to roles that should read audit lines.
  • Show HR how to open the audit tab on the employee.

What HR does

  • Review history before approving sensitive changes.
  • Print the audit PDF report when a manager asks for evidence.

Link to next phase

When data and documents are stable, employees request time off in the Leaves phase.

Phase 5 — Leaves and movements

hr_uae_leaves

Purpose

You manage UAE-style leave types: annual, special, medical, unpaid. Unpaid leave can follow an auto-deduction rule. You also track movement (where the employee is) and send alerts for vacation or permit dates.

What you set up

  • Review default leave types. Change names if your policy uses other words.
  • Train managers on approval steps in standard Odoo time off.
  • Explain the Movement Tracking report to security or HR operations.

What HR and managers do

  • Employees create leave requests. Managers approve or refuse.
  • HR checks movement reports when many people travel.

Link to next phase

Long leave can affect payroll (hold rules). Flight benefits are handled in Flight tickets.

Phase 6 — Flight tickets

hr_uae_flights

Purpose

The company records flight tickets using the expenses app. The system can create expenses for tickets. If the employee pays for a ticket, that cost can flow into payroll later.

What you set up

  • Check product data for ticket types (installed with the module).
  • Explain the process: who creates the ticket record, who approves expenses.

What HR does

  • Register tickets and link them to the correct employee.
  • Use the flight report for reconciliation with finance.

Link to next phase

Ticket costs and leave dates feed payroll calculations in the next phase.

Phase 7 — Payroll

hr_uae_payroll (+ community payroll)

Purpose

You run UAE salary structures and rules. The suite adds logic for salary hold during vacation (on-hold state and return flag). You get a payroll dashboard and a live dashboard view for analysis.

What you set up

  • Confirm structures and salary rules match your company contract text.
  • Map contracts to the correct structure for each employee group.
  • Train payroll staff on hold during vacation if you use it.

What payroll staff does

  • Create batches or payslips per period.
  • Check computed lines before confirmation.
  • Use dashboards to find exceptions (for example many absences).
Note. If you upgrade code, test the live dashboard after deploy. A past SQL issue in the live dashboard was fixed; always verify in a copy of the database first.

Link to next phase

After normal pay works, you add salary adjustments for one-time or recurring changes.

Phase 8 — Salary adjustments

hr_uae_salary_adjustment

Purpose

You record allowances and deductions outside the main contract line. Each record needs HR Manager approval (single stage). Modes: one payment, from–to date range, or recurring.

What you set up

  • Define who holds the HR Manager role for approval.
  • Explain to finance how approved lines reach payslips.

What HR does

  • Create adjustment requests with clear reason and dates.
  • Wait for approval before expecting pay impact.

Link to next phase

When employment ends, you use termination. That step closes the contract and stops future payslips.

Phase 9 — Contract termination

hr_uae_termination

Purpose

You run a clear exit workflow: the employee file can be archived, the contract ends, future payslips are cancelled where needed, and you print a termination report.

What you set up

  • Decide who may start a termination case.
  • Align the report text with your legal / HR policy.

What HR does

  • Complete termination steps in order.
  • Keep the PDF in the employee paper file or document system.

Link to next phase

Leadership often wants summary dashboards and PDF packs. That is Phase 10.

Phase 10 — Reports and dashboards

hr_uae_reports

Purpose

You get one place for important PDF reports and dashboard screens: master data, payroll, tickets, vacations, movement tracking, termination, and audit trail.

What you set up

  • Give access to managers who need read-only insight.
  • Test each report with a small sample of real-like data.

What users do

  • Open the dashboard at the start of the week.
  • Export or print PDFs for meetings.

Phase 11 — Screen look (theme and responsive layout)

hr_uae_backend_theme + web_responsive

Purpose

The backend gets a cleaner layout and a theme studio. Web responsive helps when HR uses a tablet or a small laptop screen.

What you set up

  • Open settings linked to the theme if your team wants company colors.
  • Train users where the app menu moved (if they used plain Odoo before).

End-to-end story (one path)

This is a simple path many companies follow. Your path can differ.

Install app→ Set base + security→ Create employee + UAE master data→ Upload documents→ Leaves + movements→ Flights (if used)→ Monthly payroll→ Adjustments when needed→ Termination at exit→ Reports for leadership

At every step, audit trail helps you answer “who changed this field?”.

  • Read first: KIG7/deploy/README.md — how to copy environment files, pull images, and restore a backup.

Tip. The GitHub mirror for Docker is private. Use a deploy key or token on the server. Do not share database backups in public channels.
